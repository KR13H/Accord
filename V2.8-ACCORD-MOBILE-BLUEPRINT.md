# Accord V2.8 Blueprint: Accord Mobile (Gemini Live/Voice)

## 1. Objective
Ship a production-ready mobile layer that brings voice-first accounting workflows to Tier-1/2 SME and CA operations while preserving V2.7 institutional guarantees:
- Low-latency UX (optimistic interactions)
- Deterministic demo reset reliability
- Local-first and privacy-aware AI execution paths

Target outcome: voice-to-ledger completion in less than 20 seconds from spoken intent to posted entry draft.

## 2. Product Scope (V2.8)
### In Scope
- Mobile app shell (Capacitor iOS-first, Android-ready)
- Real-time voice capture and transcript streaming
- Gemini Live orchestration for intent extraction and conversational clarifications
- Ledger draft creation from structured voice intents
- FX-aware posting preview and compliance pre-check flags
- CA handoff queue sync with existing V2.7 endpoints

### Out of Scope (for V2.8)
- Full offline-first posting with eventual reconciliation
- On-device model hosting for large LLM workloads
- Multi-tenant white-labeling
- Deep ERP connector marketplace

## 3. System Design
## 3.1 High-Level Flow
1. User speaks intent in mobile client.
2. Audio stream is segmented and transcribed.
3. Gemini Live session extracts intent + entities (amount, currency, counterparty, tax class).
4. Mobile sends normalized command to backend orchestration endpoint.
5. Backend runs deterministic validators and compliance gates.
6. UI receives optimistic draft acknowledgment immediately.
7. Background tasks complete AI rationale, nudge text, and enrichment.

## 3.2 Core Services
- `voice_gateway_service`:
  - Session lifecycle, chunk handling, transcript ordering
- `intent_router_service`:
  - Maps transcript to canonical accounting actions
- `mobile_sync_service`:
  - Device auth, session continuity, retry-safe writes
- Existing services reused:
  - `currency_service`, `compliance_service`, `voucher_service`, `report_service`

## 3.3 API Surface (Proposed)
- `POST /mobile/voice/session/start`
- `POST /mobile/voice/session/chunk`
- `POST /mobile/voice/session/commit`
- `POST /mobile/intent/execute`
- `GET /mobile/drafts/{draft_id}`
- `POST /mobile/drafts/{draft_id}/confirm`
- `POST /mobile/drafts/{draft_id}/discard`

All new writes should remain idempotent using request fingerprints plus demo reset flush compatibility.

## 4. UX Principles
- Optimistic-first: instant UI state transition after deterministic validation pass
- Human-in-control: no autonomous posting without explicit user confirm
- Explainability: each AI suggestion must expose why it was generated
- CA-ready traceability: every action mapped to audit metadata

## 5. Delivery Plan
## Phase A: Foundations (Week 1)
- Finalize mobile auth model and token lifecycle
- Implement voice session endpoints and schema contracts
- Add telemetry fields for voice event timings

Exit criteria:
- Voice session start-to-commit works in staging
- End-to-end latency p95 for session control endpoints under 250 ms

## Phase B: Voice-to-Draft (Week 2)
- Integrate Gemini Live intent extraction
- Build intent normalizer and confidence scoring
- Implement optimistic draft ACK and background enrichment

Exit criteria:
- At least 90% structured extraction accuracy for defined test phrases
- Draft ACK p95 under 200 ms

## Phase C: Compliance and CA Handoff (Week 3)
- Add GST and blocker pre-check hooks
- Sync drafts into CA alert/evaluation workflows
- Add one-click transition from mobile draft to CA queue

Exit criteria:
- No regression on V2.7 compliance gate behavior
- 0 critical errors across 50 mixed workflow runs

## Phase D: Hardening and Demo Packaging (Week 4)
- Resilience: retries, circuit breakers, degraded-path UX
- Security pass: PII redaction, transcript retention controls
- Demo mode script for deterministic showcase

Exit criteria:
- 100% reproducible demo run
- Release checklist green across mobile + backend smoke suite

## 6. Non-Functional Targets
- Mobile interaction ACK latency: p95 under 200 ms
- Voice intent-to-draft completion: p95 under 20 s
- Error budget: less than 0.5% failed workflow runs in staging
- Concurrency: sustain 100 concurrent mobile sessions in test

## 7. Security and Compliance Guardrails
- TLS-only transport and signed session tokens
- Transcript data minimization and configurable retention windows
- Endpoint-level rate limiting for session/chunk abuse prevention
- Audit log coverage for every state transition

## 8. Test Strategy
- Unit tests:
  - Intent parsing, currency normalization, idempotency keys
- Integration tests:
  - Voice session lifecycle, draft confirmation, compliance gates
- Stress tests:
  - 100 concurrent sessions with mixed chunk arrival timing
- Demo tests:
  - Deterministic reset followed by scripted investor golden path

## 9. Rollout Strategy
- Stage 1: Internal CA power users (dogfood)
- Stage 2: Controlled partner cohort in India
- Stage 3: General availability for Pro + CA tiers

Feature flags:
- `mobile_voice_enabled`
- `gemini_live_enabled`
- `mobile_optimistic_drafts`

## 10. Risks and Mitigations
- Risk: transcript ambiguity in noisy environments
  - Mitigation: confidence threshold + clarifying follow-up prompts
- Risk: AI latency spikes
  - Mitigation: preserve optimistic path and queue enrichment fallback
- Risk: compliance false positives from incomplete entities
  - Mitigation: enforce required entity checks before final posting

## 11. Definition of Done (V2.8)
- Voice workflow is stable, observable, and audit-ready
- No critical regressions to V2.7 institutional paths
- Investor demo path reproducible in one command
- Release artifact includes benchmark report + architecture note

1. Freeze `v2.7-institutional-gold`
2. Branch `feature/v2.8-accord-mobile`
3. Scaffold mobile voice gateway routes
4. Implement phase gates and telemetry first
